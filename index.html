Strava Review: User Experience

Feb. 20, 2025

In this design sprint, I analyzed Strava's interface through the lens of HCI principles, focusing on how sensors (like GPS and accelerometers), responders (like visual feedback), displays (like maps and activity graphs), and controls (like touch inputs on the mobile app) affect user interaction during their workouts when using the app. Interviews with multiple users of different backgrounds showed us that the app was very effective in supporting fitness goals but there could still be multiple improvements in user experience design. Towards the end, the results showed that Strava should minimize cognitive load and errors for the best user experience.

Incorporating Hand Gestures to Increase Accessibility

Mar. 18, 2025

In this project, my team and I tried to enhance the accessibility of a web-based game called Jewel Jet, by incorporating intuitive hand gestures (using the MediaPipe Gesture Recognizer library) to replace traditional keyboard controls. By creating gestures that directly resembled game actions, we tried to create a inclusive gaming experience for people who aren't able to interact with keyboards (i.e. users with motor impairments). Our gestures gave the user a sense of direct manipulation because of its simplicity and connection to the actual movements of the game, thus improving usability of the game.

Designing for Discoverability: A Bookstore UI/UX Makeover

Apr. 17, 2025

In this design sprint, my team and I focused on improving the UI/UX of a bookstore website. Using Bootstrap and basic web development, we enhanced the visual hierarchy, added visual elements (book covers), and implemented elements like hover effects and filters to make the interface better. We focused on HCI principles like the Gestalt principles of similarity and grouping to create a much more visually appealing and user friendly interface. User testing allowed us to improve on our prototype to create a fully functional and visually appealing final interface.

Accessibility in Amazon for Users with Cognitive and Learning Disabilities

May. 1, 2025

In this design sprint, I analyzed Amazon's interface through the perspectives of users with cognitive and learning disabilities. Despite Amazon being one of the biggest e-commerce platforms, first person accounts showed the challenges they experienced like information overload and complex navigation. By using value-sensitive, universal, and inclusive design principles, I gave multiple suggestions to improve the current user interface so it would be able to reduce issues like cognitive load, and thus enhancing usability for a all kinds of users.

Seeing More Clearly: An AI-Powered Extension for Describing Online Images

May. 12, 2025

For this project, my team and I developed a browser extension that uses AI to generate descriptions for images that either lack alt text or just need further explanation which addressed accessibility issues for not just users with visual impairments, but all users. The user is able to click and an image and get a textual description of the image back which helps understand visual content especially ones like abstract art. We were able to create assistive technology that incorporates universal design so it can be useful to all users.
